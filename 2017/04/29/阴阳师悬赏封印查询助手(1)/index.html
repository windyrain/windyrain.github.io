阴阳师悬赏封印查询助手(1)

发布 : 2017-04-29 分类 : App实践 浏览 :

© fengyu学习

因为很多身边的小伙伴都在玩阴阳师，我也加入了非酋大军中。

如果大家都还保留着一开始玩的激情，亦或是还在追求游戏中更加优秀，肯定不会放弃每天的悬赏封印。

这是一个可以得到勾玉，碎片，加成，体力，金币的免费途径！

下面就进入正题，让我们一起来看看小助手的诞生之路

1、为什么要做一个小助手

悬赏封印：在战斗中消灭指定数量的式神（涂壁 0/10）
我们来看一下官方的提示，尽管官方提示中给出了需要封印的式神出现的探索关卡，但是并未给出每个关卡式神出现的数量。

需求就这么产生了：

  • 哪个关卡涂壁多呢？
  • 除了探索关卡，还有哪有涂壁呢？

这只是从官方的提示衍生出的需求，完整的需求应该是这样的：

  • 式神列表 （提供所有式神的列表）
  • 便捷搜索 （可以利用首字母或线索进行查询）
  • 完善查询范围 （不仅是探索关卡，而且应该覆盖御魂，妖气封印等）
  • 碎片按数量排序 （查询结果由多到少排序，提升完成效率）

2、式神列表

既然有了小目标，我们就动手把它实现吧！
式神列表需要使用到React Native的ListView组件，让我们一起来学习一番吧！

2.1、代码思路

  • 实现组件 ShikigamiListView 式神列表
  • 在constructor中定义数据源
  • cloneWithRowsAndSections 将以下数据格式转变为Listview可用的数据格式，其中 B，C 将作为 section，其对应键值作为 section 下的 row 的数据来源
    {
  "B": [

      "icon": require('../images/227.png'),
      "name": "兵俑",
      "initial": "B",
      "level": "R",
      "clue": "剑/坚甲 石化"
    }, 
    {
      "icon": require('../images/264.png'),
      "name": "白狼",
      "initial": "B",
      "level": "SR"
    }
  ],
  "C": [
    {
      "icon": require('../images/265.png'),
      "name": "茨木童子",
      "initial": "C",
      "level": "SSR"
    }
  ]
}
  • 使用ListView组件暴露的API，区块头部和行分块渲染
    • renderSectionHeader：渲染区块头部
    • renderRow：渲染行
  • 实现renderSectionHeader，将SectionId（A - Z）作为渲染值，部分字母由于没有以此为拼音首字母的式神，所以不会出现，如 A
  • 实现renderRow，按照 flex 布局对式神的头像、名字、稀有度进行布局，并将查询按钮使用 flex-end 属性居于行的末端

2.2 实现代码

以下代码尽管基本完整，但细心的朋友会发现以下几点

  • 没有import react、react-native等，这个是为了缩短篇幅，大家就自己引入吧

  • ShikigamiListViewData.get(SORTBYINITIAL) 此方法得到的数据格式与代码思路列出的json数据是一致的。这个方法的实现将在后面的文章中讲述，是业务对象暴露出的方法。

  • 没有粘贴样式代码：希望大家可以实战一下 react-native 的布局，如果对文章感兴趣的是前端工程师，自然会对这种与inline-style类似的基于flex的布局有所体会。

// 1. 实现并导出式神列表组件
export default class ShikigamiListView extends Component {
  constructor(props) {
    super();
    // 2. 定义数据源
    this.ds = new ListView.DataSource({
      rowHasChanged: (r1, r2) => r1 !== r2,
      sectionHeaderHasChanged:(s1, s2) => s1 !== s2
    });
    // 3. 使用this.ds的cloneWithRowsAndSections方法生成数据
    this.state = {
      dataSource: this.ds.cloneWithRowsAndSections(ShikigamiListViewData.get(SORTBYINITIAL))
    };
  }

  render() {
    // 4. 使用ListView暴露的api：renderSectionHeader, renderRow
    return (
      <ListView
        style={styles.container}
        dataSource={this.state.dataSource}
        renderSectionHeader={this._renderSectionHeader}
        renderRow={this._renderRow}
        >
      </ListView>

    );
  }

  // 5. 实现 renderSectionHeader
  _renderSectionHeader = (sectionData, sectionId) => {
    return (
      <View style={styles.section}>
        <Text style={styles.initial}>{sectionId}</Text>
      </View>
    );
  }

 // 6. 实现 renderRow
  _renderRow = (rowData, sectionId, rowId) => {
    let shikigami = rowData;
    return (
      <View key={'row_' + shikigami.name} style={styles.row}>
        <Image style={styles.shikigamiIcon} source={shikigami.icon}/>
        <Text style={styles.shikigamiName}>{shikigami.name}</Text>
        <Text style={styles["shikigamiLevel" + shikigami.level]}>{shikigami.level}</Text>
        <View style={styles.buttonWrapper}>
          <TouchableHighlight
            style={styles.fengyin}
            underlayColor="rgba(248, 248, 240, 0.4)"
            >
            <Text style={styles.fengyintext}>封</Text>
          </TouchableHighlight>
        </View>
      </View>
    );
  }
}
